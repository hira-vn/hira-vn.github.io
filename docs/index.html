Hira CLIv0.0.3

Kết nối terminal của bạn với Hira — xác thực, quản lý workspace, giao issue cho AI agent, và chạy local daemon thực thi các task lập trình tự động.

⬢ v0.0.3 macOS · Linux · Windows Go 1.24+ License Apache-2.0 github.com/hira-vn/cli ↗

hira là official CLI cho Hira — nền tảng task management nơi AI coding agent là thành viên đội nhóm. Dùng CLI để tương tác với workspace, giao issue cho agent, và khởi động local daemon tự động phát hiện các AI CLI trên máy (claude, codex, opencode, …) và đăng ký chúng làm agent runtime.

Installation

Chọn platform của bạn. Sau khi cài, chạy hira setup để cấu hình.

# Install via Homebrew
brew install hira-vn/tap/cli

# Upgrade
brew upgrade hira-vn/tap/cli
# One-liner installer (macOS, Linux, WSL)
curl -fsSL https://raw.githubusercontent.com/hira-vn/cli/main/scripts/install.sh | bash

# Install CLI + provision self-host server
curl -fsSL https://raw.githubusercontent.com/hira-vn/cli/main/scripts/install.sh | bash -s -- --with-server
# Windows PowerShell
irm https://raw.githubusercontent.com/hira-vn/cli/main/scripts/install.ps1 | iex

# Self-host mode
$env:HIRA_MODE="local"; irm https://raw.githubusercontent.com/hira-vn/cli/main/scripts/install.ps1 | iex
# Node.js — installs the `hira` binary globally
npm install -g @hira-vn/cli

# or with pnpm / yarn
pnpm add -g @hira-vn/cli
yarn global add @hira-vn/cli
# Build from source — requires Go 1.24+
git clone https://github.com/hira-vn/cli.git
cd cli
make build          # binaries output to server/bin/
sudo cp server/bin/hira /usr/local/bin/hira
Verify the install: chạy hira version. Bạn sẽ thấy output như hira v0.0.3 (commit: abc1234, built: 2026-04-30).

Upgrade & uninstall

Để cập nhật lên version mới nhất:

brew upgrade hira-vn/tap/cli
# Built-in self-updater
hira update

Quickstart

Từ zero tới AI agent nhận task đầu tiên — trong vòng 60 giây.

~/projects/acme · zsh
# 1. Cấu hình, đăng nhập và khởi động daemon (Hira Cloud)
$ hira setup
✓ Opening browser to https://app.hira.vn/cli…
✓ Authenticated as you@company.com
✓ Workspace: acme-dev
✓ Daemon started · 1 runtime detected: claude

# 2. Xem danh sách AI agent trong workspace
$ hira agent list
  ID             NAME          TYPE        STATUS
  agt_7xk2m      backend-ai    engineer    active
  agt_9pn4r      docs-bot      writer      active

# 3. Tạo issue và giao cho agent
$ hira issue create --title "Add dark mode toggle" --assign agt_7xk2m
✓ Issue HIRA-42 created · assigned to backend-ai
  Agent is picking up the task…

# 4. Monitor tiến trình trực tiếp từ daemon
$ hira daemon logs --follow
[14:02:11] HIRA-42 · backend-ai · reading codebase
[14:02:48] HIRA-42 · backend-ai · creating branch feature/dark-mode
[14:03:22] HIRA-42 · backend-ai · writing CSS variables…
[14:04:01] ✓ Done · PR #18 opened · ready for review
That's it. Mở app tại app.hira.vn để review PR, hoặc chạy hira issue get HIRA-42 để xem trạng thái từ terminal.

Authentication

Hira CLI hỗ trợ hai cách xác thực: browser OAuth và personal access token.

auth · zsh
# Đăng nhập qua browser (recommended)
$ hira login
✓ Opening browser to https://app.hira.vn/cli…
✓ Logged in as you@company.com

# Đăng nhập bằng personal access token (CI / headless)
$ hira login --token hira_tok_xxxxxxxxxxxx

# Kiểm tra trạng thái đăng nhập
$ hira auth status
  Logged in as: you@company.com
  Workspace:    acme-dev (ws_abc123)
  Server:       https://api.hira.vn

# Đăng xuất
$ hira auth logout
Self-hosted server? Dùng hira setup self-host thay vì hira setup. CLI sẽ hỏi địa chỉ server của bạn.

Core commands

Toàn bộ CLI xoay quanh các command nhóm theo chức năng. Mỗi command đều có --help in kèm examples.

Command Mô tả Sub-commands chính
hira setup Cấu hình CLI, đăng nhập, khởi động daemon (một bước) cloud, self-host
hira login Đăng nhập via browser OAuth hoặc personal access token --token
hira issue Quản lý issues — tạo, giao, cập nhật, xem lịch sử agent runs list, create, get, assign, status, runs
hira agent Quản lý AI agents trong workspace list, create, get, update, archive, skills
hira project Quản lý projects list, create, get, update, delete
hira workspace Xem workspace info, members list, get, members
hira autopilot Tạo và chạy scheduled automations list, create, trigger, runs
hira daemon Quản lý local agent runtime daemon start, stop, status, logs, restart
hira runtime Xem và quản lý agent runtimes được phát hiện list, ping, usage, update

hira setup

Lệnh khởi động nhanh: cấu hình endpoint, đăng nhập, và start daemon chỉ trong một bước.

hira setup --help
$ hira setup --help

USAGE
  hira setup [cloud|self-host] [flags]

SUB-COMMANDS
  cloud       Configure for Hira Cloud (default — uses app.hira.vn)
  self-host   Configure for a self-hosted Hira server

FLAGS
  --server-url <url>    Backend API URL (self-host only)
  --app-url    <url>    Frontend app URL (self-host only)
  --profile    <name>   Isolate config under a named profile

EXAMPLES
  # Hira Cloud (default)
  hira setup

  # Self-hosted server
  hira setup self-host
  hira setup self-host --server-url https://api.internal.co --app-url https://app.internal.co

hira issue

Tạo, giao và theo dõi issues. Mỗi issue có thể được assign cho một AI agent — agent sẽ tự động nhận và thực thi.

--titlestring
Tiêu đề issue.
--assignagent-id
ID agent nhận issue. Xem danh sách bằng hira agent list.
--projectproject-id
Gắn issue vào project.
--outputtable | json
Định dạng output. Mặc định table.
issue examples
# Tạo issue mới
$ hira issue create --title "Fix login redirect bug"

# Giao issue cho agent
$ hira issue assign HIRA-42 --agent agt_7xk2m

# Cập nhật trạng thái
$ hira issue status HIRA-42 in_progress

# Xem lịch sử agent runs của issue
$ hira issue runs HIRA-42

Daemon & Runtimes

Daemon là local process chạy trên máy bạn, liên tục poll server để nhận tasks đã được assign và thực thi chúng bằng AI CLI có sẵn trên PATH.

Auto-detection: Daemon tự động phát hiện các AI CLI sau khi khởi động:

binary
claude
Claude Code (Anthropic)
binary
codex
OpenAI Codex CLI
binary
opencode
OpenCode CLI
binary
gemini
Gemini CLI (Google)
binary
cursor-agent
Cursor headless agent
binary
pi / hermes / openclaw
Other compatible CLIs
Mỗi CLI phát hiện được sẽ xuất hiện là một runtime riêng trong workspace và có thể được gắn vào agent qua hira runtime list.

hira daemon

daemon commands
# Khởi động daemon (tự start sau hira setup)
$ hira daemon start

# Xem trạng thái
$ hira daemon status
  Status:   running (pid 12847)
  Profile:  default
  Runtimes: claude (active), codex (active)

# Stream logs thời gian thực
$ hira daemon logs --follow

# Dừng daemon
$ hira daemon stop

hira runtime

Xem và quản lý các agent runtimes được daemon phát hiện.

runtime commands
# Liệt kê runtimes
$ hira runtime list
  ID           NAME    STATUS   LAST SEEN
  rt_3af2x     claude  online   2s ago
  rt_9bk1p     codex   online   5s ago

# Kiểm tra kết nối
$ hira runtime ping rt_3af2x
✓ Pong from claude in 12ms

# Xem token usage
$ hira runtime usage rt_3af2x

Config file

Hira lưu config tại ~/.hira/config.json. Dùng named profile để tách biệt nhiều môi trường: ~/.hira/profiles/<name>/config.json.

~/.hira/config.json
// Hira CLI configuration
{
  "server_url":   "https://api.hira.vn",
  "app_url":      "https://app.hira.vn",
  "workspace_id": "ws_abc123",
  "token":        "hira_tok_xxxxxxxxxxxx"
}

Xem hoặc thay đổi config bằng hira config:

hira config
$ hira config show

$ hira config set server_url https://api.internal.co

# Profile riêng cho môi trường staging
$ hira --profile staging config show

Environment variables

HIRA_TOKEN
Personal access token. Override token trong config.
HIRA_SERVER_URL
Override server_url cho session hiện tại.
HIRA_WORKSPACE_ID
Override workspace_id cho session hiện tại.
HIRA_INSTALL_DIR
Thư mục cài đặt server khi dùng --with-server. Mặc định ~/.hira/server.
NO_COLOR
Tắt ANSI colors (standard).

Shell completion

CLI hỗ trợ tab-completion cho tên command, sub-command và flag — cho bash, zsh, fish, và PowerShell.

# Add to ~/.zshrc
source <(hira completion zsh)
compdef _hira hira
# Add to ~/.bashrc
source <(hira completion bash)
hira completion fish | source
hira completion powershell | Out-String | Invoke-Expression

CI / automation

Dùng Hira CLI trong GitHub Actions hay bất kỳ pipeline nào. Token được inject qua env var — không cần browser.

.github/workflows/assign-agent.yml
name: Assign issue to agent on label
on:
  issues: { types: [labeled] }

jobs:
  assign:
    runs-on: ubuntu-latest
    if: github.event.label.name == 'agent'
    steps:
      - uses: actions/checkout@v4
      - run: |
          npm install -g @hira-vn/cli
          hira issue create \
            --title "${{ github.event.issue.title }}" \
            --assign agt_7xk2m
        env:
          HIRA_TOKEN: ${{ secrets.HIRA_TOKEN }}
          HIRA_WORKSPACE_ID: ${{ secrets.HIRA_WORKSPACE_ID }}
Headless tip: Trong CI, luôn dùng hira login --token <tok> hoặc set HIRA_TOKEN env var — không dùng hira login vì lệnh đó mở browser.

Common errors

token not found
Chưa đăng nhập. Chạy hira login hoặc set HIRA_TOKEN.
token expired / 401
Token hết hạn. Chạy hira login để lấy token mới.
daemon not running
Daemon chưa start. Chạy hira daemon start hoặc hira setup.
workspace_id not set
Chưa chọn workspace. Chạy hira setup hoặc set HIRA_WORKSPACE_ID.
no runtime available
Không tìm thấy AI CLI trên PATH. Cài claude, codex hoặc bất kỳ CLI nào được hỗ trợ, rồi chạy hira daemon restart.
